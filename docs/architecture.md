# هيكلة النظام — Saber Jo Snack

## الفكرة العامة
GitHub (تخزين الكود) → Render (تشغيل الكود لايف: API + قاعدة بيانات لاحقاً) → واتساب/إنستغرام/ماسنجر (قناة التواصل مع الزبون) ← webhook → موديل Claude (يفهم ويرد) → تُسجَّل الطلبات بالداشبورد.

## تدفق رسالة زبون
1. الزبون يبعت رسالة على واتساب.
2. المزود (Green API أو Meta Cloud API) يبعتها لـ `/webhook`.
3. `webhook.js` يتأكد المحادثة مش موقوفة يدوياً (`pausedConversations.json`).
4. إذا الرسالة كلمة إيقاف/استئناف، يعالجها مباشرة بدون ما يوصل الموديل.
5. غير هيك، `promptBuilder.js` يبني البرومبت النهائي من `settings.json` + `menu.json` + `deliveryFees.json`.
6. `ai.js` يبعت البرومبت + سجل المحادثة لـ Claude ويرجع الرد.
7. `whatsapp.js` يبعت الرد للزبون.
8. سجل المحادثة يتحدث بـ `conversations.json` (آخر 20 رسالة بس، توفيراً بالتوكينز).

## الصلاحيات
- **owner (مدير كامل):** كل شي — البرومبت، الأسعار، الموظفين.
- **orders (موظف طلبات):** الطلبات + توفر الأصناف بس.
- **viewer (مشاهدة فقط):** يشوف الطلبات بدون تعديل.

## نقاط توسّع لاحقة
- نقل التخزين من JSON لقاعدة بيانات (Postgres على Render).
- ربط رسمي بـ Meta WhatsApp Cloud API بدل Green API.
- تسجيل عدد مرات بيع كل صنف تلقائياً من `orders.json` بدل ما يكون رقم ثابت.
- إضافة قناة إنستغرام وماسنجر (نفس بنية الـ webhook، بس extractIncomingMessage مختلف لكل منصة).
